ES6 异步编程之生成器函数

# 一、简述

在 ES6 标准中，提供了 Generator 函数（即“生成器函数”），它是一种异步编程的解决方案。

生成器函数也是一个是函数，可以使用函数的方法进行定义与调用。

下面是一个简单的生成器函数。

// function* 为生成器函数字面量
function* generatorForPrint() {
  yield console.log('first step');
  yield console.log('second step');
  yield console.log('third step');
}

const generator = generatorForPrint(); // 调用后控制台无输出
generator.next(); // 'first step'
generator.next(); // 'second step'
generator.next(); // 'third step'

我们可以看到，按照普通函数进行调用生成器函数，并不会如同普通函数一般调用里面的内容，而是需要通过调用函数调用的返回值里的 next() 方法，才可以执行 yield 后面的打印内容。

# 二、Generator 函数语法

在上面的代码中我们可以看出来，生成器函数在执行时能暂停，后面又能通过调用返回值的 next() 从暂停处继续执行。这一点与迭代器实现简直一模一样。

在调用生成器函数时，函数内部不会直接执行它里面的代码，而是返回一个这个生成器的 迭代器 （ iterator (opens new window) ）对象 （称 生成器对象）。因此即使一个只有打印输出的生成器函数，也必须通过调用生成器函数内部的 next() 方法才可以执行。

function* generatorForPrint() {
  console.log('There is the first line of code');
  console.log('There is the second line of code');
}
const generator = generatorForPrint();
generator.next();
// 'There is the first line of code'
// 'There is the second line of code'

对比上面两个例子，不难发现 yield 可以使迭代器停止到当前其所在行的末尾，即每次调用生成器对象的 next() 方法，可以使迭代器指针从上一次所指向的位置，移动到 yield 所在位置的末尾，两次位置其中的所有代码都会被执行。

next()方法返回一个对象，这个对象包含两个属性：value 和 done，value 属性表示本次 yield 表达式的返回值，done 属性为布尔类型，表示生成器后续是否还有 yield 语句，即生成器函数是否已经执行完毕并返回。

function* generator(param) {
  console.log('first');
  yield param;
  console.log('second');
  yield param + 10;
  console.log('third');
}

const gen = generator(10);

const firstReturn = gen.next(); // first
console.log(firstReturn); // { value: 10, done: false }

const secondReturn = gen.next(); // second
console.log(secondReturn); // { value: 20, done: false }

const thirdReturn = gen.next(); // third
console.log(thirdReturn); // { value: undefined, done: true }

在 next() 的返回值中，value 的值与 yield 后面的返回值相关，即yield (opens new window) 后紧跟迭代器要返回的值。或者如果用的是 yield* (opens new window)（多了个星号），则表示将执行权移交给另一个生成器函数（当前生成器暂停执行）。

在上面这个迭代器函数中，yield 虽然只有两个，但是结合 yield 对 next() 调用时迭代器指针的偏移，明细调用第二次 next() ，第二个 yield 后面代码是不能够被执行到的，因此需要再次执行 next()。

迭代器指针在最后一个 yeild 时，调用 next() 调用后，会从最后一个 yield 的位置开始执行，直到生成器函数的内部代码块结束。且返回值开始发生变化，由于最后一次 yield 结束地方并不存在 yield 提供返回值给 value ，因此 value 自然为 undefined 。而 done 也由前面统一的返回 false ，进而转变为 true。

done 属性为布尔类型。表示生成器后续是否还有 yield 语句即生成器函数是否已经执行完毕并返回。若该函数被执行完，则 done 的值为 true 。所以当 done 为 true 时，该生成器函数就代表执行完毕，除了生成器函数从头到尾依次执行使 done 的值变成 true 之外，生成器函数内部有 return 也可以达到相同的效果。不同于 yield 隐式返回，return会显示返回并结束函数，同普通函数一样。

function* yieldAndReturn() {
  yield 'Y';
  return 'R'; //显式返回处，可以观察到 done 也立即变为了 true
  yield 'unreachable'; // 不会被执行了
}

var gen = yieldAndReturn();
console.log(gen.next()); // { value: "Y", done: false }
console.log(gen.next()); // { value: "R", done: true }
console.log(gen.next()); // { value: undefined, done: true }

对于 next() 的返回值 value ，是由 yield 后面的表达式决定的。

function* generator(name, age) {
  yield name.length >= 2 && name.length <= 4
    ? '姓名符合规范'
    : '姓名不符合规范';
  yield age >= 0 && age <= 150 ? '年龄符合规范' : '年龄不符合规范';
}

const gen = generator_2('张三', 18);
console.log(gen.next().value); // 姓名符合规范
console.log(gen.next().value); // 年龄符合规范

生成器函数除了在调用时可以接收参数，在调用 next()方法时也可以传递参数。next()方法如果传入了参数，那么这个参数会传给 上一条执行的 yield 语句左边的变量 ，例如下面例子中的 x：

function* gen() {
  x = yield 'foo';
  yield x;
}

var gen_obj = gen();
console.log(gen_obj.next()); // 执行 yield 'foo'，返回 'foo'
console.log(gen_obj.next(100)); // 将 100 赋给上一条 yield 'foo' 的左值，即执行 x=100，返回 100
console.log(gen_obj.next()); // 执行完毕，value 为 undefined，done 为 true
